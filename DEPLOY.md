# Инструкция по деплою на GitHub Pages

## Автоматический деплой

Проект настроен для автоматического деплоя на GitHub Pages при каждом push в ветку `main`.

### Настройка GitHub Pages

1. Перейдите в настройки репозитория: `Settings` → `Pages`
2. В разделе "Source" выберите:
   - **Source**: `GitHub Actions`
3. Сохраните изменения

### Как это работает

- При каждом push в ветку `main` автоматически запускается GitHub Action
- Проект собирается (build)
- Результат публикуется на GitHub Pages
- Приложение будет доступно по адресу: `https://egor51.github.io/capital/`

### Ручной запуск деплоя

Если нужно запустить деплой вручную:
1. Перейдите в раздел `Actions` репозитория
2. Выберите workflow "Deploy to GitHub Pages"
3. Нажмите "Run workflow"

## Локальная разработка

Для локальной разработки используйте:

```bash
npm run dev
```

Приложение будет доступно на `http://localhost:5173`

## Сборка для продакшена

Для локальной сборки:

```bash
npm run build
```

Собранные файлы будут в папке `dist/`

## Важно

- Base path настроен на `/capital/` для продакшена
- В локальной разработке используется `/`
- Все пути в коде должны быть относительными
